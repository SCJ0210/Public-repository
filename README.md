# Public Repository

This repository documents the setup process, completed steps, issues encountered, and solutions while learning to use Cursor IDE and GitHub.

### Cursor IDE

Installed the following extensions:

- Claude Code extension
- Codex extension

## Steps Completed

1. Installed **Cursor IDE**.
2. Opened **File → Open Editor Window**.
3. Pressed **Ctrl + Shift + X** to open the **Extensions** panel.
4. Searched for and installed the following extensions:
  - Claude Code
  - Codex
5. Created a GitHub account and a new public repository named **Public-repository**.
6. Selected **<> Code** on GitHub and downloaded the repository as a ZIP file to the local machine.
7. Returned to Cursor and selected **File → Open Folder**.
8. Opened the extracted repository folder.
9. Compiled the contents of this document in Notepad and used ChatGPT to generate a properly formatted `README.md` file with headings, lists, and sections.
10. Copied ChatGPT results and pasted it into the "Preview" editor of the README.md file opened in Cursor.
11. Saved changes and used Cursor Composer (Ctrl + I) to ask help publishing changes.
12. Downloaded Git for proper cloning and publishing.
13. Cloned Public repository locally.
14. Used Cursor Composer to commit and push changes on the public repository.

## Issues Encountered

### 1. Lack of Familiarity with Cursor IDE

- Could not find the **Extensions** panel from the initial opening window.
- Was unsure how to install extensions.

### 2. Lack of Familiarity with GitHub Workspace

- Was not sure how to open a GitHub repository in Cursor.

---

## Solutions

### Cursor IDE Extensions

- Researched online how to create a GitHub repository and install extensions in Cursor.
- Investigated possible reasons why the **Extensions** panel was unavailable from the opening window.
- Found an AI Overview result that suggested opening an **Editor Window** first and installing extensions from there.
- Successfully installed both the Claude Code and Codex extensions.

### Opening a GitHub Repository

- Searched online for instructions on opening a GitHub repository in Cursor.
- Learned that the repository must first be downloaded or cloned to the local machine.
- Opened the extracted repository folder in Cursor and successfully accessed the workspace.

### Pushing Changes to the GitHub Repository

- Used **Cursor Composer** (Ctrl + I) to ask help doing this.
- Found out that downloading it as ZIP is not enough and installing Git is required to successfuly clone the repository and commit changes.
- Downloaded Git and cloned the repository locally.
- Used the composer to commit and push changes successfully.
